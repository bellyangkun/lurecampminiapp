# 露营地小程序 · 系统说明与扩展规划

> 文档对应项目：`lurecampminiapp`（乡悦华亭度假村微信小程序）
> 关联 H5：`https://lurecamp1.xiabebe.cn`
> 后端 API：与 H5 共享同一套 Node.js 服务

---

## 一、当前系统说明

### 1.1 项目定位

面向露营/度假村场景的微信小程序，为用户提供：
- 营地内地图导航
- POI（目标点）浏览与路线规划
- 拍照打卡与集章
- 活动/餐饮/酒店预约
- 优惠券领取与使用
- AI 客服咨询

### 1.2 技术栈

| 层级 | 技术 |
|-----|------|
| 前端 | 微信小程序原生框架（WXML + WXSS + JS） |
| 地图 | 腾讯地图 `map` 组件，GCJ-02 坐标系 |
| 后端 | Node.js 共享 API（与 H5 同一份服务） |
| 登录 | 微信 code + 后端 jscode2session，支持手机号验证码登录 |
| 数据存储 | 后端 SQLite / JSON 文件（由 server 决定） |

### 1.3 目录结构

```

│   ├── checkin/        # 我的打卡：印章统计、最近打卡缩略图
│   ├── booking/        # 活动预约：可预约列表、我的预约、预约表单
│   ├── coupons/        # 优惠券：可领取、我的券

│   ├── me/             # 我的：用户信息、统计入口
│   ├── login/          # 登录页（微信登录 / 手机号登录）
│   ├── profile-setup/  # 编辑资料
```

### 1.4 功能模块详情

#### 1.4.1 地图导航（pages/index）

- 全屏腾讯地图，显示营地 POI
- POI 按类型分类：入口、活动、服务、快闪、餐厅、卫生间、酒店、茶馆、其他
- 分类 chip 过滤
- 点击 POI  marker 选择目标，自动画虚线路线、计算距离和步行时间
- 支持唤起微信内置地图导航
- 自定义分类 marker 图标（`images/markers/`）
- 选中目标显示名称 callout
- 底部「拍照打卡」「定位到我」快捷按钮

#### 1.4.2 拍照打卡（pages/index + pages/checkin）

- 点击底部「拍照打卡」唤起相机
- 自动寻找 200m 内最近 POI
- 拍照后走 `/api/ar/shoot` 合成 AR 贴图
- 提交打卡 `/api/checkins`，累积印章
- 「我的打卡」页显示：印章总数、累计打卡次数、最近打卡列表
- 点击缩略图可预览大图

#### 1.4.3 活动预约（pages/booking）

- 「可预约」tab：按类型筛选活动/餐饮/酒店/主题项目
- 「我的预约」tab：查看个人预约记录及状态（待确认/已确认/已取消）
- 预约表单：姓名、手机、日期、人数、备注
- 提交后等客服确认

#### 1.4.4 优惠券（pages/coupons + pages/me）

- 可领取优惠券列表
- 「我的券」tab 显示已领券
- 券码支持一键复制
- 「我的」页面显示可用优惠券数量

#### 1.4.5 AI 客服（pages/ai）

- 调用后端 `/api/ai` 接口
- 提供营地相关问答

#### 1.4.6 我的（pages/me）

- 用户头像、昵称、会员标识
- 数据卡片：已打卡、优惠券、我的预约
- 快捷入口：我的打卡、优惠券、我的预约、AI 客服
- 呼叫前台、用户协议、退出登录

#### 1.4.7 登录与身份（utils/user.js）

- 未登录时生成匿名 userId（设备级稳定）
- 微信 code 登录或手机号验证码登录
- 登录后合并匿名 userId 数据
- 退出登录保留设备级 userId，便于找回数据

## 二、扩展可能性

以下按业务价值和实现复杂度分为短期、中期、长期三个阶段。

### 2.1 短期可落地（1-2 周）

#### 2.1.1 餐饮在线点单

- 扫码或从餐厅 POI 进入点单页
- 菜单分类、购物车、在线支付
- 订单状态：待接单、制作中、可取餐、已送达
- 价值：直接增收，减少前台排队

#### 2.1.2 活动日历

- 按日期展示营地活动：篝火晚会、露天电影、亲子手工、草坪音乐会
- 活动详情页 + 一键报名/预约
- 已报名活动提醒
- 价值：提升活动参与率，增加二次消费

#### 2.1.3 主题路线推荐

- 在地图上推荐固定路线：亲子路线、徒步路线、宠物友好路线、拍照打卡路线
- 选择路线后高亮显示沿途 POI
- 价值：降低游客决策成本，提升体验

#### 2.1.4 分享海报

- 打卡成功后生成精美海报（带照片、地点、印章数）
- 支持保存图片和分享到朋友圈
- 价值：自然拉新，社交传播

### 2.2 中期建设（1-2 月）

#### 2.2.1 积分商城

- 打卡、消费、分享获得积分
- 积分兑换咖啡、烧烤券、装备租赁时长、升级房型
- 价值：提升用户粘性和复购

#### 2.2.2 会员等级体系

- 银卡 / 金卡 / 钻卡会员
- 不同等级享折扣、生日特权、优先预约
- 价值：沉淀高价值用户

#### 2.2.3 游客相册墙

- 用户打卡照片公开展示（可选）
- 点赞、精选、按 POI/活动筛选
- 价值：UGC 内容，增强社区氛围

#### 2.2.4 停车导航与缴费

- 显示营地停车场位置及剩余车位
- 导航到最近空位
- 出场扫码缴费
- 价值：解决节假日停车痛点

#### 2.2.5 装备租赁

- 帐篷、天幕、桌椅、炊具线上预约
- 到店自取或配送到营位
- 价值：延伸消费场景

### 2.3 长期探索（3 月以上）

#### 2.3.1 AR 实景导航

- 结合摄像头 + 指南针，在实景画面上叠加箭头指引
- 适合营地小路、树林等复杂环境
- 价值：差异化体验，降低迷路概率

#### 2.3.2 AI 导游讲解

- 到达 POI 后自动触发语音/文字讲解
- 介绍历史、玩法、注意事项
- 价值：替代人工导游，提升沉浸感

#### 2.3.3 智能行程规划

- 用户输入：人数、天数、偏好（亲子/徒步/美食/拍照）
- AI 自动生成营地内行程：路线、活动、餐饮、住宿
- 价值：提升预订转化率

#### 2.3.4 IoT 硬件联动

- 预订帐篷/房车后手机蓝牙开锁
- 营地 WiFi 一键连接
- 水电桩、充电桩实时状态
- 价值：提升科技感，减少人工成本

#### 2.3.5 营地社群

- 按入住日期/营位建群
- 活动组队、二手交易、失物招领
- 价值：增强用户归属感和复购

## 三、推荐优先级

如果资源有限，建议按以下顺序推进：

1. **餐饮在线点单 + 活动日历** → 直接带来收入，开发成本低
2. **主题路线推荐 + 分享海报** → 提升体验 + 拉新
3. **积分商城 + 会员等级** → 留存和复购
4. **停车导航与缴费** → 解决高峰痛点
5. **AR 导航 / AI 导游** → 打造差异化亮点

---

## 四、技术注意事项

- 新增页面需要在 `app.json` 中注册
- 涉及支付的模块需提前申请微信支付商户号
- 地图相关功能注意坐标系统一（后端 WGS-84，小程序 GCJ-02）
- 图片/文件上传需在微信小程序后台配置 `uploadFile` 合法域名
- 登录、手机号授权需符合微信小程序隐私协议要求
